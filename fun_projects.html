Simulation of Belousov Zhabotinsky reaction

One of the projects that I am very proud of is a Java program that simulates Belousov Zhabotinsky reactions using cellular automata. The reason why I am very proud of this project is because it is my first computer simulation and I did not think that it was going to work properly. Before moving to the coding aspect of this project, I want to talk about Belousov Zhabotinsky reactions.

Belousov Zhabotinsky reactions are amazingly complicated and a great example of a moshpit of fields in STEM. From the very first time it was presented to the world; physicist, chemists, and biologists have been fixated on trying to determine a mechanism behind the reaction. Knowing basic chemistry, this type of reaction is freaky! Imagine you are doing a titration and see the indicator go off, but then a few moments later the indicator disappears and you have a transparent solution again! This type of behavior would indicate that your titration is going in reverse, which is not thermodynamically possible, yet this is exactly what you observe in a Belousov Zhabotinsky reaction!

Now, Belousov Zhabotinsky reactions are not breaking physics, they are instead an example of physics in conditions in which most people are not taught about. In the case of Belousov Zhabotinsky reactions, these reactions take place in conditions in which the chemical system is far away from thermal equilibrium. In a Belousov Zhabotinsky reaction there are two chemicals that interact with each other. This interaction can be described with two sets chemical reactions in which chemical A is depleted while chemical B is increased and vice verse. During these chemical interactions, the energy available to the system is decreasing, so these chemical reactions happen until the energy available is completely depleted (thermal equilibrium).

The code for this simulation is not as complicated as the mechanism behind the Belousov Zhabotinsky reaction. This project was created during my CS321 class and I actually wrote it as an extension for a main project. The main project had us using cellular automata to create a program that simulates Conway's Game of Life. The simulation of the Belousov Zhabotinsky reaction is a lot like the program for Conway's Game of Life; the Game of Life has special kinds of cells, and those cells behave differently depending on their environment (i.e a cell can only be alive if it has a certain number of neighbors that are alive). In the Belousov Zhabotinsky reaction, each cell would represent some part of a medium containing concentrations of three different chemicals. For each time step, all the cells will follow a chemical reaction involving all three chemicals, and the concentrations will be calculated using the average calculation within a cell's neighborhood. This is a lot like Conway's Game of Life, except now, we are using a continuous form of cellular automata as opposed to a discrete form.
